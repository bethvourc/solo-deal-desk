# Solo Deal Desk 🤝

**Turn one discovery-call transcript into an auditable, contract-ready deal package — in a single run.**

## What it does

Paste a sales-call transcript (or lead email thread) and Solo Deal Desk produces the
entire "turn a conversation into signed revenue" workflow that a real company splits
across a sales rep, solutions engineer, pricing analyst, legal, and finance:

1. **Qualification scorecard** — the FIT-BUST rubric (6 dimensions, /18) with a clear
   Pursue / Caution / Decline call and the single biggest risk to closing.
2. **Tailored proposal** — your prospect's pain in their words, scope + out-of-scope, and
   three value-anchored tiers (Good / Better / Best) with a recommendation.
3. **SOW / contract draft** — deliverables, milestones, payment terms, revisions, IP-on-
   final-payment, kill fee, late fees (with a legal-review disclaimer).
4. **Deterministic invoice schedule** — generated by a zero-dependency Python script whose
   line items **provably sum to the contract total**. Verifiable, not vibes.
5. **Timed follow-up sequence** — four send-ready messages (Day 0 → break-up), each doing
   one job.
6. **Evidence ledger** — the audit trail that maps major proposal and pricing claims back
   to the source conversation, so unsupported claims become `[NEEDS CONFIRMATION]`.
7. **CRM update + sendable email** — a strict JSON pipeline update and the actual short
   email the founder can send to move the deal forward.

Everything lands in `deal-packages/<client>/` with an index README and a checklist of every
fact the prospect didn't confirm, so you never send a proposal with a made-up budget.

## The OPC pain point it solves

A solo founder's scarcest hour is the one right after a good sales call. That's when the
deal is hot — and when, alone, you have to qualify, price, write a proposal, draft a
contract, and set up follow-ups before momentum dies. Most solos either stall for days or
ship a sloppy, generic proposal. Solo Deal Desk collapses that hour of cross-functional
work into one grounded, verifiable run — without inventing facts, overpromising, or losing
the thread between what the prospect said and what you send.

## How to use it

1. Install the skill into your agent (BotLearn / Claude-style Agent Skill format).
2. (Optional) Copy `founder-profile.example.md` to `founder-profile.md` and fill in your
   name, business, offer, default rate, currency, and payment preferences so the docs come
   out personalized.
3. Give your agent the conversation and ask it to run Solo Deal Desk. For example:

   > Run Solo Deal Desk on this discovery call: \<paste transcript or attach
   > `examples/sample-discovery-call.md`\>

4. Review the **scorecard**, give a go/no-go, and the agent builds the full package.
5. Resolve any `[NEEDS CONFIRMATION]` items, then send.

### Try it in 30 seconds

```bash
# See the verifiable pricing engine on its own:
python3 scripts/invoice_schedule.py --total 14000 --currency USD \
    --deposit-pct 40 --milestones 3 --start 2026-06-24 --cadence-days 7

# Run the package smoke check:
python3 scripts/demo_check.py
```

A fully worked example — input transcript and the complete generated package — lives in
[`examples/`](examples/). For a judge-friendly walkthrough, use
[`examples/run-demo.md`](examples/run-demo.md).

## Why it wins

- **Verifiable.** The pricing math is a deterministic script — anyone can re-run it and
  confirm the numbers sum. No hallucinated totals.
- **Auditable.** The evidence ledger maps proposal, pricing, and follow-up claims back to
  the source conversation. Unsupported facts surface as confirmation items.
- **Revenue-oriented.** It tackles the fragile post-call window where solo founders leak
  deals by waiting too long or sending generic, under-scoped proposals.
- **Runnable day one.** Pure-stdlib script, no API keys, no setup. Works on a stranger's
  machine immediately.

> The contract output is a **draft**, not legal advice. Have counsel review before signing.
